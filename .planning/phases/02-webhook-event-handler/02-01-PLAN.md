Create `app/modules/webhooks/signature.py`:
- Function `verify_signature(payload_body: bytes, signature_header: str, webhook_secret: str) -> bool`
- Calendly sends the signature in the `Calendly-Webhook-Signature` header
- The header format is: `t=<timestamp>,v1=<signature>` (there may be multiple v1 values)
- To verify: compute HMAC-SHA256 of `<timestamp>.<payload_body>` using the webhook signing key
- Compare the computed signature against the v1 value(s) using `hmac.compare_digest`
- Return True if any v1 value matches, False otherwise
- If the header is missing or malformed, return False (don't raise)
- Use structlog for debug/warning logging
- Reference: https://developer.calendly.com/api-docs/ZG9jOjM2MzE2MDM4-webhook-signatures

Create `app/modules/webhooks/parser.py`:
- Dataclass `WebhookEvent` with fields:
  - `event_type: str` (e.g., "invitee.canceled" or "invitee.no_show")
  - `scheduled_event_uri: str` (full URI like "https://api.calendly.com/scheduled_events/UUID")
  - `event_name: str` (e.g., "AHG Payments Discovery")
  - `invitee_email: str`
  - `invitee_uri: str` (full invitee URI for dedup key)
  - `calendly_event_uuid: str` (extracted UUID from scheduled_event_uri)
  - `is_discovery: bool` (True if event_name contains "Discovery")
  - `is_onboarding: bool` (True if event_name contains "Onboarding")
  - `raw_payload: dict` (original payload for DLQ/debugging)

- Function `parse_webhook_payload(body: dict) -> WebhookEvent`:
  - Extract from Calendly webhook structure:
    - `body["event"]` -> event_type
    - `body["payload"]["scheduled_event"]["uri"]` -> scheduled_event_uri
    - `body["payload"]["scheduled_event"]["name"]` -> event_name
    - `body["payload"]["email"]` -> invitee_email
    - `body["payload"]["uri"]` -> invitee_uri
  - Extract UUID from scheduled_event_uri (last path segment)
  - Compute is_discovery and is_onboarding from event_name (case-insensitive contains)
  - Store full body as raw_payload
  - Raise ValueError with descriptive message if required fields are missing

- Function `filter_event(event: WebhookEvent) -> bool`:
  - Return True if event.is_discovery or event.is_onboarding
  - Return False otherwise (caller will log and skip)
  </action>

Create `app/modules/webhooks/router.py`:
- Create an APIRouter with tag="webhooks"
- POST `/calendly` endpoint:
  1. Read raw body bytes from request (`await request.body()`)
  2. Get `Calendly-Webhook-Signature` header from request
  3. Get webhook secret from `request.app.state.settings.calendly_webhook_secret`
  4. Get slack_client from `request.app.state.slack_client`
  5. **Always return JSONResponse(status_code=200)** — wrap entire handler in try/except (EVNT-09)
  6. Verify signature using `verify_signature()`. If invalid:
     - Log warning with structlog: "webhook_signature_invalid"
     - Send Slack alert: ":warning: Atlas: Calendly webhook received with INVALID signature. Payload rejected."
     - Return 200 with `{"status": "rejected", "reason": "invalid_signature"}`
  7. Parse JSON body and call `parse_webhook_payload(body)`. If ValueError:
     - Log error: "webhook_parse_error"
     - Return 200 with `{"status": "rejected", "reason": "parse_error"}`
  8. Call `filter_event(event)`. If False:
     - Log info: "webhook_filtered", event_name=event.event_name, event_type=event.event_type
     - Return 200 with `{"status": "filtered", "reason": "not_discovery_or_onboarding", "event_name": event.event_name}`
  9. If event passes filter:
     - Log info: "webhook_accepted", event_type=event.event_type, event_name=event.event_name, email=event.invitee_email
     - For now, return 200 with `{"status": "accepted", "event_type": event.event_type, "event_name": event.event_name}` (matching + field writes come in Plans 02-02 and 02-03)
  10. Any unhandled exception in the try block:
     - Log error: "webhook_processing_error" with exc_info
     - Send Slack alert: ":x: Atlas: Webhook processing error: {str(e)}"
     - Return 200 with `{"status": "error", "reason": str(e)}`

Update `app/main.py`:
- Import the webhook router: `from app.modules.webhooks.router import router as webhooks_router`
- Add `app.include_router(webhooks_router, prefix="/webhooks")` after the existing route definitions
