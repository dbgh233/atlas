Dataclass `FieldWriteResult`:
- `success: bool`
- `fields_written: list[dict]` — list of `{"field_name": "...", "field_id": "...", "value": "..."}` dicts
- `error: str | None`

Async function `write_field_updates(ghl_client: GHLClient, match_result: MatchResult, event: WebhookEvent) -> FieldWriteResult`:
1. Determine the appointment type from `match_result.appointment_type` (GHL is source of truth). If None, fall back to event classification (event.is_discovery -> "Discovery", event.is_onboarding -> "Onboarding").
2. Determine the field updates based on event_type + appointment_type:

   | event_type | appointment_type | Fields to write |
   |------------|-----------------|-----------------|
   | invitee.no_show | Discovery | Discovery Outcome = "No Show", Appointment Status = "No-Show" |
   | invitee.no_show | Onboarding | Appointment Status = "No-Show" |
   | invitee.canceled | Discovery | Appointment Status = "Cancelled" |
   | invitee.canceled | Onboarding | Appointment Status = "Cancelled" |

3. Build the GHL update payload:
   ```python
   custom_fields = [{"id": field_id, "field_value": value} for field_id, value in updates]
   payload = {"customFields": custom_fields}
   ```
   Note: GHL API uses "field_value" not "value" in the customFields array.

4. Call `ghl_client.update_opportunity(match_result.opportunity_id, payload)`
5. Return FieldWriteResult with the fields written
6. On GHL API error: return FieldWriteResult(success=False, fields_written=[], error=str(e))

Use structlog for all logging: "field_write_start", "field_write_success", "field_write_error".

**Create `app/modules/webhooks/notifications.py`:**

All functions take `slack_client: SlackClient` as first arg and are async.

Function `notify_webhook_success(slack_client, event: WebhookEvent, match_result: MatchResult, write_result: FieldWriteResult)`:
- Message: ":white_check_mark: Atlas: Webhook processed successfully\n- Event: {event.event_type}\n- Contact: {event.invitee_email}\n- Opportunity: {match_result.opportunity.get('name', 'Unknown')} (ID: {match_result.opportunity_id})\n- Match Method: {match_result.match_method}\n- Fields Updated: {comma-separated field names}"

Function `notify_webhook_match_failure(slack_client, event: WebhookEvent, reason: str)`:
- Message: ":mag: Atlas: Webhook received — no matching opportunity\n- Event: {event.event_type}\n- Contact: {event.invitee_email}\n- Event Name: {event.event_name}\n- Reason: {reason}"

Function `notify_webhook_error(slack_client, event: WebhookEvent | None, error: str)`:
- Message: ":x: Atlas: Webhook processing error\n- Error: {error}\n- Event: {event.event_type if event else 'unknown'}\n- Contact: {event.invitee_email if event else 'unknown'}"

Function `notify_webhook_filtered(slack_client, event: WebhookEvent)`:
- Message: ":fast_forward: Atlas: Webhook filtered (not Discovery/Onboarding)\n- Event Name: {event.event_name}\n- Event Type: {event.event_type}"
- Note: This is informational, low priority. Consider making this a debug-level log instead if Slack gets noisy. For now, send it.

Function `notify_signature_invalid(slack_client)`:
- Message: ":warning: Atlas: Calendly webhook with INVALID signature rejected"

All notification functions should catch exceptions internally (log error, don't re-raise) — notification failure should never break webhook processing.

**Rewrite `app/modules/webhooks/router.py` with full pipeline:**

The POST `/calendly` endpoint becomes the complete pipeline:

```
receive -> verify signature -> parse -> filter -> idempotency check -> match -> write fields -> notify
```

Full flow (replace the current implementation):

1. Read raw body and signature header
2. Always return 200 (wrap everything in try/except)
3. **Verify signature** — if invalid: notify_signature_invalid(), return 200 rejected
4. **Parse payload** — if parse error: notify_webhook_error(), return 200 rejected
5. **Filter event** — if filtered: notify_webhook_filtered(), return 200 filtered
6. **Idempotency check** — compute idempotency key as `f"calendly:{event.event_type}:{event.invitee_uri}"` (invitee_uri is unique per invitee per event). Check `IdempotencyRepository(db).exists(key)`. If exists: log "webhook_duplicate", return 200 with `{"status": "duplicate"}`
7. **Match opportunity** — if no match: notify_webhook_match_failure(), add to DLQ (DLQRepository), return 200 match_failed
8. **Write field updates** — call `write_field_updates(ghl_client, match_result, event)`
9. **Record idempotency** — if write succeeded: `IdempotencyRepository(db).add(key, event.event_type, "success")`. If write failed: add(key, event.event_type, "write_error") and add to DLQ
10. **Notify** — call notify_webhook_success() on success, notify_webhook_error() on write failure
11. Return 200 with result

Get `db` from `request.app.state.db` (this is the aiosqlite.Connection from lifespan).

**Add admin endpoint for subscription setup:**

Add a POST `/admin/webhooks/setup` endpoint to the router (or create a separate admin router — keep it simple, add to webhooks router with tag="admin"):
1. Get calendly_client from `request.app.state.calendly_client`
2. Get the callback URL from request body JSON: `{"callback_url": "https://atlas-production-248a.up.railway.app/webhooks/calendly"}`

4. Return the result
