  <action>
Modify `write_field_updates()` in field_writer.py to accept two new parameters:
- `dry_run: bool = False` — when True, skip the GHL API call entirely
- `slack_client: SlackClient | None = None` — needed for verification failure alerts

Add `verified: bool | None` and `verification_details: str | None` fields to `FieldWriteResult`.

**Dry-run path:** When `dry_run=True`:
1. Build the payload exactly as before (custom_fields list)
2. Log the payload with structlog: `log.info("dry_run_write_skipped", opp_id=..., payload=payload)`
3. Return `FieldWriteResult(success=True, fields_written=fields_written, verified=None)` — success=True because the "write" conceptually succeeded in dry-run, verified=None because no write to verify.

**Read-back verification path:** When `dry_run=False` and the write succeeds:
1. After the existing `await ghl_client.update_opportunity(...)` call succeeds
2. Call `opp = await ghl_client.get_opportunity(match_result.opportunity_id)`
3. Extract the opportunity's customFields array from `opp.get("customFields", [])`
4. For each field written (field_id, expected_value), find the matching customField by id and compare its `value` (GHL returns `value` on reads, even though writes use `field_value`)
5. If ALL fields match: set `verified=True` on the result
6. If ANY field mismatches or customFields is missing: set `verified=False`, set `verification_details` to a string describing the mismatch (e.g., "Discovery Outcome: expected 'No Show', got 'Scheduled'")
7. On verification failure, call `await notify_verification_failure(slack_client, match_result.opportunity_id, verification_details)` — wrap in try/except so notification failure doesn't break the pipeline
8. If the get_opportunity call itself fails (exception), set `verified=False`, `verification_details="Read-back failed: {error}"`, log the error, send Slack alert, but do NOT change `success=True` (the write itself succeeded)

In notifications.py, add `notify_verification_failure(slack_client, opp_id, details)`:
```python
async def notify_verification_failure(
    slack_client: SlackClient,
    opp_id: str,
    details: str,
) -> None:
    try:
        await slack_client.send_message(
            f":warning: Atlas: GHL write verification FAILED\n"
            f"- Opportunity: {opp_id}\n"
            f"- Details: {details}\n"
            f"- Action: Check GHL opportunity manually"
        )
    except Exception as e:
        log.error("notify_verification_failure_failed", error=str(e))
```
